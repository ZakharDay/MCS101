Структура проекта

Как организовать свой код

Я предлагаю следующую логику

reset.css Сбрасывает браузер
basics.css Настраевает базовые элементы
layout.css Описывает стилистику главных элементов шаблона
content.css Описывает контент

Контент — это абстрактное название, называйте свои описания контента понятными для вас названиями. В нашем дизайне у нас есть фото и видео секции, которые логически отличаются, но в целом могут быть похожи. Отсюда можно выделить структуру: photo-section.css, video-sections.css, в которых можно описывать соответствующие свойства. Какие-то смежные свойства можно описывать в layout.css, а какие-то фундаментальные свойства в basics.css.

Хороший принцип для любого кодера — не повторять, а переиспользовать код или по другому DRY.

Параметр overflow

Как положить в блок больше чем он может в себя вместить

Свойства

overflow Определяет что будет если контент внутри бока по размеру будет больше чем сам блок
overflow-x Определяет что будет если контент внутри бока по ширине будет больше чем сам блок
overflow-y Определяет что будет если контент внутри бока по высоте будет больше чем сам блок

Значения

visible Покажет всё, что выходит за границы элемента
hidden Спрячем всё, что выходит за границы элемента
scroll Появится скроллинг (полоса прокрутки). В зависимости от указанного свойства и размера внутреннего блока, может быть скролл по горизонтали, по вертикали или оба сразу.
auto Скроллы будут показыватся при неободимости
initial Сбрасывает значение на значение по умолчанию
inherit Будет насдедовать значение родителя

Forms, Inputs, Textareas

Формы на сайте

Вообще, формы сложная штука, потому что вы передаёте через интернет какие-то данные и иногда эти данные являются конфиденциальной информацией, но это вопрос к back-end разработчикам и специалистам по защите информации.

Что касается стилизации, в целом, всё то же самое что и с обычными блоками. Есть два правила, которые вам могут помочь.

textarea, input { outline: none; }
textarea { resize: none; }
